イベント処理

イベント処理は、クリックや入力などの出来事をきっかけに JavaScript を動かす仕組みです。Webページに「反応」を付ける中心になる考え方です。

addEventListener

イベント処理では、「何が起きたら」「どんな処理をするか」を結び付けます。 `addEventListener()` はそのための基本の書き方です。 

const button = document.querySelector("#save-button");

button.addEventListener("click", function () {
  console.log("ボタンが押されました");
});

よく使うイベント

  • `click`: クリックしたとき
  • `input`: 入力内容が変わったとき
  • `change`: 値の変更が確定したとき
  • `submit`: フォーム送信時
const input = document.querySelector("#user-name");

input.addEventListener("input", function () {
  console.log(input.value);
});

クリックで画面を変える例

イベント処理は DOM 操作と組み合わせることで、ページに反応を付けられます。

const button = document.querySelector("#change-button");
const message = document.querySelector("#message");

button.addEventListener("click", function () {
  message.textContent = "ボタンが押されました";
});

学習のポイント

  • 「何が起きたら」「何をするか」の形で考えると分かりやすいです
  • まずは `click` と `input` から覚えると実践しやすいです
  • イベント処理は DOM 操作とセットで使う場面が多いです
要素を取得する前に JavaScript が実行されると、イベントを付けたい要素がまだ存在しないことがあります。HTML の読み込み順も意識すると安心です。
