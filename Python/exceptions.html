例外処理

実行中に起こるエラーに備える仕組みが例外処理です。入力ミスやファイルの存在確認などで特に役立ちます。

try と except

例外処理を使うと、エラーが出てもそこで強制終了せず、利用者に分かりやすいメッセージを出したり、代わりの処理をしたりできます。 

try:
    number = int(input("数字を入力してください: "))
    print(10 / number)
except ValueError:
    print("整数を入力してください")
except ZeroDivisionError:
    print("0では割れません")

else と finally

  • `else`: 例外が起きなかったときに実行
  • `finally`: 例外の有無に関係なく最後に実行
try:
    file = open("sample.txt", "r", encoding="utf-8")
except FileNotFoundError:
    print("ファイルが見つかりません")
else:
    print(file.read())
finally:
    print("処理を終了します")

学び始めのポイント

  • 全部のエラーをまとめて扱うより、原因ごとに分けると分かりやすいです。
  • まずは `ValueError` と `FileNotFoundError` から覚えると実用的です。
  • 例外処理は「エラーを隠す」ためではなく、「安全に処理する」ために使います。

よくある例外

  • `ValueError`: 型変換できない値を渡したとき
  • `ZeroDivisionError`: 0で割ろうとしたとき
  • `FileNotFoundError`: ファイルが見つからないとき
  • `IndexError`: 存在しない添字を参照したとき
items = [10, 20]

try:
    print(items[5])
except IndexError:
    print("その位置のデータはありません")

例外処理を使う場面

ユーザー入力、ファイル読み込み、API通信など、「うまくいかない可能性がある処理」で特に役立ちます。 失敗を前提にして書くことで、利用者にも分かりやすいプログラムになります。
