アルゴリズムと計算量

アルゴリズムは問題を解く手順、計算量はその手順がどれくらい効率的かを考える指標です。データ量が増えたときの速さを考えるうえで重要です。

線形探索の例

線形探索は、先頭から順番に1つずつ探す方法です。 人名一覧を上から見ていって、目的の名前を探すイメージに近いです。 

def linear_search(items, target):
    for index, item in enumerate(items):
        if item == target:
            return index
    return -1

先頭から順に見るため、要素数に比例して時間がかかりやすいです。

なぜ計算量を考えるのか

データが少ないときは違いが見えにくくても、件数が増えると処理時間に大きな差が出ることがあります。 計算量は、その増え方をざっくり比較する考え方です。

計算量の見方

計算量は「データが増えたとき、処理の大変さがどう増えるか」を表す目安です。 最初は正確な式よりも、増え方の違いをざっくり比べられれば十分です。

  • `O(1)`: 一定時間
  • `O(n)`: データ数に比例
  • `O(n^2)`: 二重ループなどで増えやすい
for a in items:
    for b in items:
        print(a, b)
`O(n^2)` はデータが少ないうちは問題なくても、件数が大きくなると急に重くなりやすい考え方です。

学習のポイント

最初は厳密な証明よりも、「データが大きくなると、どの処理が重くなるか」を意識すると理解しやすいです。

  • 二重ループは重くなりやすいと覚える
  • リスト検索と辞書検索の違いに注目する
  • まずは読みやすさと必要十分な速さの両立を目指す
