クラスとオブジェクト指向

クラスは、データと処理をひとまとめにする設計図です。似た性質を持つものを整理して扱いたいときに役立ちます。

クラスの基本

クラスから作られた具体的な実体をオブジェクトと呼びます。 最初は「同じ形のデータを何回も使いたいときの設計図」と考えると分かりやすいです。 `__init__` はオブジェクトを作るときに最初の値を入れる場所です。 

class Student:
    def __init__(self, name, grade):
        self.name = name
        self.grade = grade

    def introduce(self):
        print(f"{self.name}です。{self.grade}年生です。")


student1 = Student("Soshi", 2)
student1.introduce()

`Student("Soshi", 2)` のように書くと、`name` と `grade` を持つ1人分のデータが作られます。

属性とメソッド

オブジェクトが持つデータを属性、オブジェクトに対して行う処理をメソッドと呼びます。 `self` は「今操作しているそのデータ自身」を指しています。 

class Book:
    def __init__(self, title):
        self.title = title
        self.is_borrowed = False

    def borrow(self):
        self.is_borrowed = True


book = Book("Python入門")
book.borrow()
print(book.title)
print(book.is_borrowed)
  • `self.title` は本の名前というデータです
  • `borrow()` は本の状態を変える動きです
  • 「データ」と「そのデータに関する操作」を近くに置けるのがクラスの良さです

オブジェクト指向の考え方

  • データとそれを扱う処理を近くに置ける
  • 似た機能を持つものをまとめやすい
  • 大きなプログラムを役割ごとに分けやすい
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount


account = BankAccount("Soshi")
account.deposit(5000)
print(account.balance)

学び始めのポイント

最初は「関連するデータと関数を1つにまとめる」感覚で十分です。無理に複雑な設計を覚えるより、小さな例で慣れるのがおすすめです。

  • まずは `__init__` とインスタンス生成に慣れる
  • 関数だけでは整理しづらい場面でクラスを試す
  • いきなり継承まで広げず、小さな設計から始める
最初の段階では、クラスを「難しい仕組み」と考えすぎなくて大丈夫です。 生徒、商品、本のように、似た情報をまとめたい場面で使う箱だと思うと理解しやすくなります。
