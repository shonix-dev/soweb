仮想環境と pip

仮想環境は、プロジェクトごとにPythonのライブラリを分けて管理する仕組みです。`pip` はライブラリをインストールするための道具です。

なぜ必要か

プロジェクトごとに使うライブラリやバージョンが違うと、同じPC上で衝突することがあります。仮想環境を使うと、それぞれを分離できます。

基本コマンド

python3 -m venv .venv
source .venv/bin/activate
pip install requests
pip freeze > requirements.txt
  • `venv` で仮想環境を作成
  • `activate` で有効化
  • `pip install` でライブラリ追加
  • `requirements.txt` に依存関係を書き出し

よくある作業の流れ

mkdir my-project
cd my-project
python3 -m venv .venv
source .venv/bin/activate
pip install flask requests
pip freeze > requirements.txt

別の環境で再現するときは、仮想環境を作ってから `pip install -r requirements.txt` を実行します。

学習のポイント

新しいプロジェクトでは、まず仮想環境を作る習慣を付けると安全です。特にチーム開発では `requirements.txt` の共有が重要になります。

  • グローバル環境には必要最小限だけ入れる
  • プロジェクトごとに独立した環境を使う
  • README にセットアップ手順を書くと再利用しやすいです
