モジュールとパッケージ

Pythonでは、処理をファイル単位やフォルダ単位で整理できます。小さく分けて管理することで、再利用しやすく、見通しのよいコードになります。

モジュールとは

1つの `.py` ファイルをモジュールとして使えます。関数や定数を別ファイルに切り出し、`import` で読み込みます。

# math_tools.py
def add(a, b):
    return a + b


# main.py
import math_tools

print(math_tools.add(3, 5))
コードを用途ごとに分けると、修正箇所が見つけやすくなります。

パッケージとは

複数のモジュールをフォルダ単位でまとめたものがパッケージです。機能が増えてきたときに整理しやすくなります。

project/
├─ main.py
└─ tools/
   ├─ __init__.py
   ├─ file_utils.py
   └─ text_utils.py
from tools.file_utils import load_text

content = load_text("memo.txt")
print(content)

import の書き方

モジュールの読み込みにはいくつかの書き方があります。書き方でコードの見通しや名前の扱い方が変わります。

import math
from math import sqrt
import math_tools as mt

print(math.pi)
print(sqrt(16))
print(mt.add(2, 3))
  • `import module` は、どこから来た処理か分かりやすいです。
  • `from module import name` は短く書けます。
  • `as` を使うと、別名を付けて読みやすくできます。

使い分けの考え方

  • 1ファイルで十分ならモジュール
  • 複数機能を分類したいならパッケージ
  • 共通処理は早めに切り出すと保守しやすいです

最初は「長くなったコードを別ファイルに分ける」だけでも十分です。プログラムが大きくなってきたら、 機能ごとにフォルダを分ける意識を持つと整理しやすくなります。
