APIの基本利用

APIは、他のサービスやシステムとデータをやりとりするための窓口です。Pythonでは `requests` を使ったHTTP通信がよく登場します。

基本の流れ

APIは、プログラムどうしが決まったルールでやり取りする仕組みです。 最初は「URLにお願いを送って、返ってきたデータを使う」とイメージするとつかみやすいです。

  • URLにリクエストを送る
  • レスポンスを受け取る
  • JSONなどを解析して使う
import requests

url = "https://api.example.com/users"
response = requests.get(url, timeout=10)

if response.status_code == 200:
    data = response.json()
    print(data)

`requests.get()` が「取りに行く処理」、`response.json()` が「受け取った内容をPythonで使いやすい形にする処理」です。

POST のイメージ

取得だけでなく、データを送信するときにもAPIを使います。フォーム送信や新規登録では `POST` がよく登場します。

payload = {"name": "Soshi", "score": 90}
response = requests.post(
    "https://api.example.com/scores",
    json=payload,
    timeout=10,
)

`payload` に送信したいデータをまとめ、`json=payload` としてAPIに渡しています。

よく見る要素

  • HTTPメソッド: `GET`, `POST` など
  • ステータスコード: `200`, `404`, `500` など
  • 認証: APIキーやトークン
実際にAPIを使うときは、公式ドキュメントで利用条件や認証方法を確認します。

初学者向けの見方

  • まずは `GET` で取得して `json()` を読むところから覚える
  • 通信失敗時の処理も考える
  • 認証情報はコードに直接書かず、環境変数などで管理するのが安全です
if response.status_code == 200:
    print("成功")
else:
    print("失敗", response.status_code)

最初は「成功したか」「失敗したか」を表示できるだけでも十分です。少しずつ例外処理や認証へ広げていきましょう。
