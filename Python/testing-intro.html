テスト入門

テストは、プログラムが期待どおり動くかを確認するための仕組みです。手で毎回確認するより、コードとして残すことで再利用できます。

unittest の基本

import unittest

def add(a, b):
    return a + b

class TestAdd(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(2, 3), 5)


if __name__ == "__main__":
    unittest.main()

何をテストするか

まずは、入力に対して出力が決まる小さな関数から始めると分かりやすいです。

  • 正常系の入力
  • 境界値の入力
  • 想定外入力に対する振る舞い
def is_adult(age):
    return age >= 18


class TestAge(unittest.TestCase):
    def test_is_adult(self):
        self.assertTrue(is_adult(18))
        self.assertFalse(is_adult(17))

テストを書くメリット

  • 修正後の動作確認がしやすい
  • 不具合の再発を防ぎやすい
  • 仕様の理解をコードで残せる

手動確認だけに頼るより、あとからの機能追加やリファクタリングにも強くなります。
