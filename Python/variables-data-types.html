変数とデータ型

Pythonでは、値に名前を付けて保存するときに変数を使います。さらに、その値が整数なのか文字列なのかといった種類をデータ型と呼びます。

基本

変数は「あとで使う値を入れておく箱」のようなものです。Pythonでは、変数名を書いて `=` で値を代入するだけで使い始められます。

  • 変数は `=` を使って代入します。
  • Pythonでは型を書く必要はなく、代入した値から自動で判断されます。
  • よく使う型は `int`、`float`、`str`、`bool` です。
name = "Soshi"
age = 20
height = 170.5
is_student = True

print(name)
print(age)
print(type(height))

`type()` を使うと、その値がどのデータ型なのか確認できます。

主なデータ型

型が違うと、できる操作も変わります。数値なら計算、文字列なら文字の結合が代表例です。

  • `int`: 整数
  • `float`: 小数
  • `str`: 文字列
  • `bool`: `True` か `False`
数値の `10` と文字列の `"10"` は別物です。見た目が似ていても、計算できるかどうかが変わります。 
print(10 + 5)      # 15
print("10" + "5")  # 105

型変換

`input()` で受け取った値は文字列になるため、計算に使う前に整数へ変換する場面がよくあります。 

text_number = "25"
number = int(text_number)

print(number + 5)   # 30
print(str(number))  # "25"

学習のポイント

  • 変数名は役割が伝わる名前にすると読みやすくなります。
  • 型の違いを意識すると、エラーの原因を見つけやすくなります。
  • まずは `print()` と `type()` を使って確認しながら学ぶのがおすすめです。

変数を使う場面

変数は、入力された値や途中計算の結果を一時的に保存するときに使います。何度も使う値に名前を付けることで、 コードの意味が分かりやすくなります。 

price = 1200
tax_rate = 0.1
total = price * (1 + tax_rate)

print(total)

つまずきやすい点

  • `score` と `Score` は別の変数です。
  • 文字列と数値をそのまま足そうとするとエラーになります。
  • 意味のない短い変数名ばかりだと、あとで読み返しにくくなります。
age = "20"

# print(age + 1)  # エラーになる
print(int(age) + 1)
